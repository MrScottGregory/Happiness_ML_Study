Model Details & Objectives

Random Forests (RF) and Gradient Boosting Machine (GBM) both are ensemble learning methods and predict, Regression or Classification, by combining the outputs from individual trees. The purpose of using this model was to classify what features are important for the happiness index we were studying in and to compare these with the results of the random forest.

Underlying gradient boosting, is the idea of additive modeling. The idea is that we are going to add a bunch of simple terms together to create a more complicated expression. In gradient boosting, that expression (function) represents a model mapping some observation's feature, x, to a target value, y. However, Instead of creating a single powerful model, boosting combines multiple simple models into a single composite model. The idea is that, as we introduce more and more simple models, or weak learners, the overall model becomes stronger and stronger.

Data Cleaning

In addition to dropping null values, and renaming variables for simplify, columns that were related, or inter-associated to one another were dropped to avoid multicollinearity and overfitting our models. When preparing the testing and training data, for gradient boost models we had to use the standard scaler to pass our values in the right format to the model.

Conclusion

When creating the gradient boost model, the number of stages and the learning rate, are hyperparameters that affect model accuracy. The more stages we use, the more accurate the model, but the more likely we are to be overfitting. The primary value of the learning, is to reduce overfitting of the overall model

With the parameters chosen, we got our model to a .90 accuracy score when running our testing data.

When running this model, economic indicators, like average wages and GDP had a higher weight in predicting happiness.
